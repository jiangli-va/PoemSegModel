## 训练数据构造：词典生成候选+LLM重排

### 文件结构

```cmd
\Segmentation
├── README.md
├── 古汉语词典及词频5以上bigram总结表v9.xlsx
├── run_llm_segmentation.py   # 主入口。输入：仅包含古诗文本的input.txt；输出：seg_llm.csv
├── utils_tries.py            # 读取词典表格，构建前缀树Trie
├── read11w.py                # 读取大规模的xlsx文件，提取词语转写至dict_all.txt, 供后续构建Trie使用
├── config.py                 # 候选词打分：根据多个特征加权算出一个浮点数score。
├── candidate_segmenter.py    # 遍历所有可能的切分组合, 进行音步奖励，结合词典得分给出topK候选
└── llm_rerank.py             # 调用LLM从候选切分中选择最佳切分
```

### 主要流程

- 构建数据`untils_tries.py`
  - 从`cyac`库导入`Trie`类，构建前缀树以支持高效的词典查询。
  - 读取Excel表格中的词典数据并生成字典树
- 词典打分`config.py`
  - 对Trie树中的每一个词进行打分，综合考虑频度、MI、词典数、是否典故和词长偏好
    - 基础分：平滑频度`score = math.log1p(freq_val)`
    - MI加成：归一化`mi_norm = (mi_val - mi_min) / (mi_max - mi_min + 1e-6)`，加成`score += 0.8 * mi_norm`
    - 词典收录加成：`score += 0.3 * cnt_val`
    - 典故加成：`score += 1.0 if is_allusion else 0.0`
    - 词长偏好：`if len == 2: score += 0.5; elif len == 3: score += 0.2; elif len >= 4: score -= 0.2`
- 音步奖励+返回候选`candidate_segmenter.py`
  - 去除标点并判断句型
  - DFS遍历所有切分组合，计算词典得分+音步奖励
    - 五言诗：2+3. 若切分点在第二个字后面，奖励0.8分；在第三个字后面，惩罚0.3分
    - 七言诗：2+2+3. 若切分点在第二、四个字后面，奖励0.5分；在第三、第五个字后面，惩罚0.3分
    - 词长微调：奖励双音词(+0.05)，惩罚单音词(-0.05)。
  - 返回topK候选切分组合
- LLM重排`llm_rerank.py`
  - 构造prompt，调用LLM进行重排
    - 注入领域法则(G-seg规范): 
      ```python
      【G-Seg 分词规范（请严格遵守）】
        0. 总体原则：根据句意；应分尽分；实事求是；合并词、分短语。如“江南|岸”，而非“江南岸”，“楼上|雁”而非“楼上雁”，“浑|不动”而非“浑不动”。
        1. **功能词、虚词（如“或、又、无、兮、其、乃、遂、也、矣”等副词、连词、语气词、否定词）应尽量单独成词。**
        2. 语义紧密、习惯搭配或专有名词的词组可以合并成多字词，如“含胎”“十月”“山河”“江南”“黄阁”“东篱”“空山”“新雨”“草木”（“一” + 量词，“不” + 动词，两个近义词并列）等。
        3. 有些地方的分词可能存在多种合理的方案，请你选择最符合句意表达的那一项。如“一尊|酒” “一|尊酒”都可行，则根据具体句意选择更合适的方案。
        4. 不允许增删、更改任何汉字，只比较不同的切分位置；不可机械追求最长词或最多单字，每个词包含字数一般最好不超过3个。
        5. **诗句里面重要的地名、人名、专有名词、固定短语，应当合并成词**，如“黄鹤楼”，“司马相如”，“鲍参军”，“韩荆州”，“长亭”，“灞桥”，“寒山寺”，“瀼西市”，“白帝城”等。
      ```
    - Few-shot示例：提供一些切分示例，展示正确的切分方式和不合理的切分方式
      ```python
      【示例一】
        原句：春风又绿江南岸
        候选：
        A. 春风|又绿|江南|岸
        B. 春风|又绿江|南岸
        C. 春风|又|绿|江南|岸
        最优答案：C
        
      【示例二】
        原句：或作金兮或似雪
        候选：
        A. 或作|金兮|或似雪
        B. 或作|金兮或|似雪
        C. 或|作|金|兮|或|似|雪
        最优答案：C
      ```
    - 限定输出格式：要求大模型“只输出一个大写字母编号”。
    - 容错机制：允许大模型在无法判断的情况下输出"WARNNING"，并提供自己的切分建议
